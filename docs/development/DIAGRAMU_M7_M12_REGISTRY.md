# M7-M12 schemų registry

> Tikslas: viena prižiūrima vieta M7-M12 schemų routing, interaktyvumo, layout SOT ir testų būsenai. Naujos schemos turi būti planuojamos per šį registry prieš keičiant `modules.json` arba React komponentus.

## Image key policy

- Nauji `sections[].image` raktai turi būti semantiniai: `m7_*`, `m9_*`, `m10_*`, `m12_*`.
- Legacy SVG keliai (`/da_pipeline_6.svg`, `/da_bi_schema_4.svg`) lieka palaikomi per `diagramRenderers.tsx` alias, bet naujiems įrašams nenaudojami.
- Jei schema moko procesą, žingsnius arba turi LT/EN tekstus, ji turi būti React komponentas per `EnlargeableDiagram`; statinis assetas tinka tik iliustracijai.
- Interaktyvi schema turi naudoti `InteractiveDiagramShell` ir `DiagramStepNav`; SVG hit zonos yra tik pointer-only per `DiagramStepHitArea`.

## Santykis su M1-M9 auditu

M7-M9 etalonas ir vizualinis smoke backlog'as aprašytas `docs/development/DIAGRAMU_M1_M9_AUDITAS.md`. Šis registry yra aktyvus M7-M12 routing, test coverage ir naujų `sections[].image` raktų pratęsimas; naujos M10+ schemos registruojamos čia prieš keičiant `modules.json`.

## Priėmimo vartai

1. Registry eilutė atnaujinta prieš pridedant arba keičiant `sections[].image` raktą.
2. `diagramRenderers.tsx` turi React rendererį visiems procesiniams M7-M12 `image` raktams.
3. Interaktyvios schemos turi testą, kuris tikrina `nav button` skaičių ir kad SVG neturi `role="button"` / `tabIndex`.
4. Dark mode patikra turi įrodyti, kad schema naudoja tamsų SVG foną arba sąmoningai dokumentuotą išimtį.
5. M7-M9 core profiliai sinchronizuojami per `npm run generate:core-data`, jei normalizuojami full SOT `image` raktai.
